Foreword

首先，整理这些知识点来自于知乎问题：Android开发中，有哪些让你觉得相见恨晚的方法、类或接口？ 中前排回答 StephenLee、macro hard、Rocko 以及其他部分回答者的答案（见谅不能一一注明）。主要的来源是 Rocko答案中提到的 Android Tips Round-Up 系列文章。本文主要整理 Cool Android Apis ，对于知乎问题中不少回答者提到的一些Android开发的技巧、第三方框架之类的不作整理。

Content

  • Activity.startActivities()

其实我们绝大多数的开发者可能是没有用过这个方法的，根据我个人理解，用的到场景并不多。除了个方法最直接的理解就是使用intent开启多个Activity，在Google的关于Activity.startActivities()文档说明中，并没有获取到出了StartActivity之外更多的信息。于是我继续扒源码，果然在在Context和ContextCompat 下找到了更加详细的说明和解释。

Context，以及 documentation of ContextCompat 

* Launch multiple new activities.  This is generally the same as calling
* {@link #startActivity(Intent)} for the first Intent in the array,
* that activity during its creation calling {@link #startActivity(Intent)}
* for the second entry, etc.  Note that unlike that approach, generally
* none of the activities except the last in the array will be created
* at this point, but rather will be created when the user first visits
* them (due to pressing back from the activity on top).

从上面我们理解到的，startActiviyies 会创建一个新的Task Stack，任务栈里Activity的位置基于我们传入的Intent数组。当我们按返回键时，就会将栈顶的Activity移除。。其实这也是framework管理用户开启的activity的方式。这里结合Activity的启动模式来理解，就简单多了。

至于应用场景，我目前能想到的就是点击通知栏来开启应用的Activities，而开启哪些Activity以及相应顺序，我们就可以用到这个方法了。

    这个方法简单粗暴，但是我没用过。这个方法会将输入的字母根据键盘上的映射转换为数字。
